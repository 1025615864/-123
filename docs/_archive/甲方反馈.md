## 一、前言

贵司于近期交付了"百姓法律助手（百姓助手）v1.0"项目源码及相关文档（包括《甲方项目功能报告（交付版）》和《技术对接报告》）。

我方在接收交付物后，依据合同约定的功能范围及业务需求，对项目进行了全面评估与测试。经评估发现，**当前交付物存在核心业务流程缺失、功能模块不完整、商业闭环无法实现等重大问题**，与预期的可运营状态存在较大差距。

## 二、核心问题清单

### 【问题一】律师服务模块严重不完整，无法形成业务闭环

**问题等级：** 🔴 严重

**问题描述：**

作为法律服务平台的核心模块，律师服务目前仅实现了律所列表展示和详情查看功能。根据贵司文档描述，律师用户角色定义为"扩展角色存在（用于律所/律师服务相关能力对接）"，实际上该角色无任何可操作功能。

**文档引用：**

> 《甲方项目功能报告》第 3.6 节："律所/律师服务（/lawfirm）——律所列表、律所详情、咨询记录（用户侧入口）"
>
> 《甲方项目功能报告》第 2 节："律师用户（lawyer）——作为扩展角色存在"

**缺失功能清单：**

| 序号 | 缺失功能          | 功能说明                             | 影响范围      |
| ---- | ----------------- | ------------------------------------ | ------------- |
| 1.1  | 律师入驻/注册流程 | 律师申请入驻平台的功能               | 律师端        |
| 1.2  | 律师认证审核流程  | 律师资质审核功能                     | 管理端        |
| 1.3  | 律师个人主页管理  | 律师编辑个人信息、服务项目、收费标准 | 律师端        |
| 1.4  | 在线预约咨询功能  | 用户预约律师咨询服务                 | 用户端        |
| 1.5  | 律师接单/拒单功能 | 律师处理咨询预约                     | 律师端        |
| 1.6  | 在线沟通工具      | 用户与律师的即时通讯/私信功能        | 用户端/律师端 |
| 1.7  | 服务订单管理      | 订单创建、状态跟踪、完成确认         | 全端          |
| 1.8  | 服务评价功能      | 用户对律师服务进行评价               | 用户端        |
| 1.9  | 律师收入与提现    | 律师查看收入、申请提现               | 律师端        |
| 1.10 | 律师端管理后台    | 律师专属的管理界面                   | 律师端        |

**我方诉求：**

请提供律师服务模块的完整功能设计方案及开发计划，确保用户可以完成"浏览律师 → 在线预约 → 付费 → 咨询服务 → 评价"的完整业务流程。

### 【问题二】支付系统仅有骨架，无法实际使用

**问题等级：** 🔴 严重

**问题描述：**

支付功能是平台商业化运营的核心基础设施。根据贵司文档说明，当前支付功能"以接口/验签骨架为主"，无法完成实际的收付款操作。

**文档引用：**

> 《甲方项目功能报告》第 7 节边界说明："支付渠道（支付宝/微信）在当前实现中以接口/验签骨架为主；实际对接需要甲方提供支付平台参数与回调规范。"

**缺失功能清单：**

| 序号 | 缺失功能          | 功能说明                                   |
| ---- | ----------------- | ------------------------------------------ |
| 2.1  | 支付渠道实际对接  | 微信支付、支付宝支付的实际接入             |
| 2.2  | 支付回调业务处理  | 支付成功后触发业务状态变更                 |
| 2.3  | 订单状态完整流转  | 待支付 → 已支付 → 服务中 → 已完成 → 已评价 |
| 2.4  | 退款流程          | 退款申请、审核、执行、回调                 |
| 2.5  | 律师/律所结算功能 | 服务收入结算与提现                         |
| 2.6  | 财务对账功能      | 平台财务报表与对账                         |
| 2.7  | 发票功能          | 用户申请发票（如需）                       |

**我方诉求：**

1. 请明确支付模块的交付边界：贵司负责完成哪些工作，哪些需要我方配合？
2. 请提供支付功能完整对接的开发计划。
3. 我方可提供支付平台（微信/支付宝）的商户参数，请贵司完成对接。

| 4.3  | 知识库数据的格式规范是什么？                   |

1. 请明确知识库初始数据的交付责任。
