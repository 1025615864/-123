# role-docs

你现在扮演文档与交付负责人。目标是让口径可追溯、入口无断链。

1. 确认本次变更涉及哪些权威文档：
   - `docs/PRD.md` / `docs/TECH_SPEC.md` / `docs/API_DESIGN.md` / `docs/DATABASE.md` / `docs/CHANGELOG.md`
2. 更新入口互链：
   - `README.md` 与 `docs/README.md` 必须能找到新增/变更文档
3. 阶段性材料归档：
   - 一次性报告/演练记录进 `docs/_archive/`
4. 收尾输出：
   - 本次文档更新点列表
   - 仍待补充的文档缺口
